# 🌾 Subsidy Scoring — Decentrathon 5.0 / Кейс 2

Система скоринга заявок на субсидии АПК Казахстана на основе машинного обучения.
Merit-based подход вместо принципа «первый подал — первый получил».

---

## 📋 Что делает система

- **Ранжирует** заявителей по вероятности одобрения (score 0–100)
- **Объясняет** каждое решение через SHAP waterfall (какие факторы повлияли)
- **Формирует shortlist** кандидатов с трёхзонной рекомендацией:
  - ✅ **Авто-одобрение** — высокий приоритет, p ≥ 0.807
  - ⚠️ **Ручная проверка** — передать комиссии, 0.55 < p < 0.807
  - ❌ **Авто-отклонение** — низкий приоритет, p ≤ 0.55

## 🧠 Логика ML модели

### Постановка задачи

**Цель:** предсказать вероятность одобрения заявки на субсидию АПК.

**Целевая переменная (v5):**
```python
# Позитивный класс — target = 1:
{"Исполнена", "Одобрена", "Сформировано поручение", "Получена"}  # 86.4%

# Негативный класс — target = 0:
{"Отклонена", "Отозвано"}  # 13.6%
```

Такой target выбран потому что "Одобрена" означает заявка прошла комиссию
и ждёт выплаты. Предсказывать только "Исполнена" несправедливо для областей
где выплаты ещё не завершились к моменту выгрузки данных.

### Архитектура ансамбля

```
Данные заявки
      ↓
Feature Engineering (33 признака)
      ↓
┌─────────────┐   ┌─────────────┐
│  LightGBM   │   │  CatBoost   │
│  5-fold OOF │   │  5-fold OOF │
│     GPU     │   │     GPU     │
└──────┬──────┘   └──────┬──────┘
       └────────┬─────────┘
          Meta LogReg (стекинг)
               ↓
       Isotonic Calibration
               ↓
        P(одобрения) ∈ [0, 1]
               ↓
        Трёхзонное решение
```

### Признаки (33 шт.)

| Группа | Описание | Кол-во |
|---|---|---|
| Временны́е | месяц, день года, день недели, час, квартал, пиковый сезон | 6 |
| Финансовые | log(сумма), log(норматив), поголовье, сумма/норматив, кратность, остаток, дециль | 7 |
| Ранг заявки | порядок подачи в день по акимату | 1 |
| Target encoding | успешность по области / акимату / району / типу / направлению / (область×направление) | 6 |
| Frequency encoding | частота встречаемости области и типа субсидии | 2 |
| Относительные суммы | сумма vs медиана по области, сумма vs медиана по типу | 2 |
| Внешние агрегаты | историческая успешность и объёмы выплат по 17 областям (data.egov.kz) | 9 |

### Оптимизация гиперпараметров

Optuna (50 trials, TPE sampler) оптимизирует **Precision @ Recall ≥ 85%**:

```python
# FP = одобрили недостойного  → дорогая ошибка (расход бюджета)
# FN = отклонили достойного   → комиссия пересмотрит (дешевле)
# Цель: максимум precision при охвате ≥ 85% достойных заявок
```

### Результаты (v5)

| Метрика | Значение |
|---|---|
| P@Recall≥85% | **0.9141** ★ |
| ROC-AUC | 0.7746 |
| PR-AUC | 0.9526 |
| Precision авто-одобрения | **91.4%** |
| Авто-одобрение | 80.4% |
| Ручная проверка | 15.4% |
| Авто-отклонение | 4.3% |
| Обучающих заявок | 36 651 |

### Топ признаки (SHAP)

1. **Успешность района** — исторический % одобрения в конкретном районе
2. **Успешность типа субсидии** — как часто одобряют данный тип
3. **Успешность (область × направление)** — комбинированный региональный сигнал
4. **День года** — сезонность: разные периоды имеют разный % одобрения
5. **Порядок подачи в день** — подтверждает эффект "первый подал — первый получил"
